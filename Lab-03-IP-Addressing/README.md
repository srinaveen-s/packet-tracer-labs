# Lab 03 – IPv4 Address Configuration

## 📌 Lab Overview

In this lab, four computers were connected to a Cisco 2960 switch. Each computer was configured with a unique IPv4 address in the same network. Connectivity was verified using the Ping command.

---

## 🎯 Objective

The objective of this lab is to understand how IPv4 addressing works in a Local Area Network (LAN) and verify communication between devices.

---

## 🖥 Network Topology

Devices Used

• 1 Cisco 2960 Switch

• 4 PCs

- Admin-PC
- HR-PC
- Finance-PC
- IT-PC

---

## 🌐 IP Address Configuration

Admin-PC

IP Address : 192.168.1.10

Subnet Mask : 255.255.255.0

---

HR-PC

IP Address : 192.168.1.20

Subnet Mask : 255.255.255.0

---

Finance-PC

IP Address : 192.168.1.30

Subnet Mask : 255.255.255.0

---

IT-PC

IP Address : 192.168.1.40

Subnet Mask : 255.255.255.0

---

## 🧪 Verification

A successful Ping test was performed between the computers.

Example

ping 192.168.1.40

The Ping reply confirmed that all devices were able to communicate successfully.

---

## 🎓 Learning Outcome

After completing this lab, I learned:

- How to configure IPv4 addresses
- How devices communicate inside a LAN
- How to verify connectivity using Ping
- Importance of correct subnet masks
- Basic switch connectivity
